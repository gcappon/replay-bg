# Initializers

# initCoreVariables

function [environment, model, mcmc, dss] = initCoreVariables(data, ip)

Initializes the core variables (i.e., environment, model, mcmc, and dss) of ReplayBG.

# Inputs

  • data: timetable (required)
    A timetable which contains the data to be used by the tool;
  • ip: InputParser (required)
    The MATLAB® input parser.

# Outputs

  • environment: structure
    A structure that contains general parameters to be used by ReplayBG;
  • model: structure
    A structure that contains general parameters of the physiological model;
  • mcmc: structure
    A structure that contains the hyperparameters of the MCMC identification procedure;
  • dss: structure
    A structure that contains the hyperparameters of the integrated decision support system.

# initEnvironment

function environment = initEnvironment(modality, saveName, saveSuffix, plotMode, verbose)

Initializes the environment core variable.

# Inputs

  • modality: vector of char (required)
    A vector of characters that specifies if the function will be used to identify the ReplayBG model on the given data or to replay the scenario specified by the given data;
  • saveName: vector of char (required)
    A vector of characters used to label, thus identify, each output file and result;
  • saveSuffix: vecotr of char (required)
    A vector of characters to be attached as suffix to the resulting output files' name;
  • plotMode: integer (required)
    A numerical flag that specifies whether to show the plot of the results or not;
  • verbose: integer (required)
    A numerical flag that specifies the verbosity of ReplayBG.

# Output

  • environment: structure
    A structure that contains general parameters to be used by ReplayBG.

# initModel

function model = initModel(data, sampleTime, glucoseModel, seed)

Initializes the model core variable.

# Inputs

  • data: timetable (required)
    A timetable which contains the data to be used by the tool;
  • sampleTime: integer (required)
    An integer that specifies the sample time of the data timetable;
  • glucoseModel: vector of char (required)
    A vector of characters that specifies the glucose model to use;
  • seed: integer (required)
    An integer that specifies the random seed. For reproducibility.

# Output

  • model: structure
    A structure that contains general parameters of the physiological model.

# initMarkovChainMonteCarlo

function mcmc = initMarkovChainMonteCarlo(maxETAPerMCMCRun,maxMCMCIterations,maxMCMCRuns, maxMCMCRunsWithMaxETA, MCMCTheta0Policy, bayesianEstimator,preFilterData, saveChains)

Initializes the mcmc core variable.

# Inputs

  • maxETAPerMCMCRun: double (required)
    A number that specifies the maximum time in hours allowed for each MCMC run;
  • maxMCMCIterations: integer (required)
    An integer that specifies the maximum number of iterations for each MCMC run;
  • maxMCMCRuns: integer (required)
    An integer that specifies the maximum number of MCMC runs;
  • maxMCMCRunsWithMaxETA: integer (required)
    An integer that specifies the maximum number of MCMC runs having maximum ETA;
  • MCMCTheta0Policy: vectro of char (required)
    A vector of characters defining the policy used by the MCMC procedure to set the initial MCMC chains values;
  • bayesianEstimator: vector of char (required)
    A vector of characters defining which Bayesian estimator to use to obtain a point estimate of model parameters;
  • preFilterData: integer (required)
    A numerical flag that specifies whether to filter the glucose data before performing the model identification;
  • saveChains: integer (required)
    A numerical flag that specifies whether to save the resulting mcmc chains in dedicated files (one for each MCMC run) for future analysis or not.

# Output

  • mcmc: structure
    A structure that contains the hyperparameters of the MCMC identification procedure.

# initDecisionSupportSystem

function dss = initDecisionSupportSystem(enableHypoTreatments,hypoTreatmentsHandler,enableCorrectionBoluses,correctionBolusesHandler)

Initializes the dss core variable.

# Inputs

  • enableHypoTreatments: integer (required)
    A numerical flag that specifies whether to enable hypotreatments during the replay of a given scenario;
  • hypoTreatmentsHandler: vector of char (required)
    A vector of characters that specifies the name of the function handler that implements an hypotreatment strategy during the replay of a given scenario;
  • enableCorrectionBoluses: integer (required)
    A numerical flag that specifies whether to enable correction boluses during the replay of a given scenario;
  • correctionBolusesHandler: vector of char (required)
    A vector of characters that specifies the name of the function handler that implements a corrective bolusing strategy during the replay of a given scenario.

# Output

  • dss: structure
    A structure that contains the hyperparameters of the integrated decision support system.
